# PageIndex QA

使用 MCP 工具进行多文档问答。

## 可用工具

| 工具 | 说明 |
|------|------|
| `list_documents` | 列出所有已索引文档 |
| `index_document` | 索引新文档 |
| `get_document` | 获取文档元数据 |
| `get_document_structure` | 获取目录结构 |
| `get_page_content` | 获取指定页面内容 |

## 必须遵守的规则

1. **必须遍历所有文档** — 不能跳过任何已索引的文档
2. **每个文档都必须查询** — 至少调用一次 get_document_structure 获取结构
3. **必须返回所有相关页面** — 问题可能涉及多个文档的多个页面

## 输出格式（必须严格按此格式）

每个来源一行，不同来源之间用空行分隔：

```
来源: {doc_name} ({type} 第 x-y 行/页)
---
{该来源对应的答案内容}

来源: {doc_name} ({type} 第 x-y 行/页)
---
{该来源对应的答案内容}
```

例如：
```
来源: readme.md (md 第 10-20 行)
---
这是关于安装步骤的说明...

来源: api.md (md 第 50-60 行)
---
API 接口使用方法如下...
```

## 禁止事项

- 不要输出 `根据xxx` 而不标注具体页面
- 不要遗漏任何文档
- 不要只查询一个文档就返回答案

## pages 格式

- `"5-7"` — 第5到7页
- `"3,8"` — 第3和第8页
- `"10"` — 仅第10页
